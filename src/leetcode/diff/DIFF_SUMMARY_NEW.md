# 差分数组算法总结

## 一、核心概念

### 什么是差分数组？

差分数组是一种用于**高效处理区间修改**的数据结构。

**定义**：对于原数组 `a`，差分数组 `d` 满足：
```
d[0] = a[0]
d[i] = a[i] - a[i-1]  (i > 0)
```

**还原**：通过前缀和还原原数组：
```
a[i] = d[0] + d[1] + ... + d[i]
```

### 为什么高效？

对区间 `[l, r]` 所有元素加 `val`：
- **暴力法**：遍历修改，O(n)
- **差分法**：只需两步，O(1)
  ```
  d[l] += val
  d[r+1] -= val
  ```

### 适用场景

- 多次区间修改操作
- 最后一次性查询结果
- 不关心中间过程

## 二、题目详解

### 2.1 基础应用

#### [1094. 拼车](./1094/1094.go)

**题目**：判断能否完成所有行程，任意时刻乘客数不能超过容量。

**思路**：每个行程 `[num, from, to]` 是一次区间修改：
- `from` 位置上车 → `diff[from] += num`
- `to` 位置下车 → `diff[to] -= num`

```go
diff := make([]int, 1002)
for _, trip := range trips {
    diff[trip[1]] += trip[0]   // 上车
    diff[trip[2]] -= trip[0]   // 下车
}
// 前缀和检查是否超容量
```

**复杂度**：O(n + m)，n 为行程数，m 为位置范围

---

#### [1109. 航班预订统计](./1109/1109.go)

**题目**：统计每个航班的预订座位总数。

**思路**：每个预订 `[first, last, seats]`：
```go
diff[first] += seats
diff[last+1] -= seats
```

**注意**：差分数组长度设为 `n+2`，避免 `last+1` 越界。

**复杂度**：O(n + m)

### 2.2 进阶应用

#### [2381. 字母移位 II](./2381/2381.go)

**题目**：对字符串进行多次区间移位操作。

**思路**：差分 + **模运算**处理环形字母表

```go
// 初始化差分数组（相邻字符差值）
diff[0] = s[0] - 'a'
diff[i] = (s[i] + 26 - s[i-1]) % 26

// 区间移位
if dir == 1 {  // 向后
    diff[start] = (diff[start] + 1) % 26
    diff[end+1] = (diff[end+1] + 25) % 26
} else {       // 向前
    diff[start] = (diff[start] + 25) % 26
    diff[end+1] = (diff[end+1] + 1) % 26
}
```

**关键点**：字母表是环形的，用模 26 处理。

**复杂度**：O(n + m)

---

#### [2406. 将区间分为最少组数](./2406/2406.go)

**题目**：将区间分成若干组，同组内区间不相交，求最少组数。

**核心洞察**：**最少组数 = 最大重叠数**

**思路**：用差分统计每个位置被多少区间覆盖

```go
for _, v := range intervals {
    diff[v[0]]++      // 区间起点
    diff[v[1]+1]--    // 区间终点后一位
}
// 前缀和的最大值即为答案
```

**复杂度**：O(n + m)

### 2.3 综合应用

#### [2528. 最大化城市的最小供电站](./2528/2528.go)

**题目**：可额外建造 k 座供电站，求所有城市最小电量的最大值。

**算法组合**：**差分 + 二分答案 + 贪心**

```go
// 1. 差分计算初始电量
for i, v := range stations {
    l, r := max(0, i-r), min(n-1, i+r)
    diff[l] += v
    diff[r+1] -= v
}

// 2. 二分答案
l, r := 0, math.MaxInt64
for l <= r {
    m := (r - l) / 2 + l
    if check(m, r, k, diff) {
        l = m + 1
    } else {
        r = m - 1
    }
}

// 3. 贪心检查：从左到右，不足则当前位置补
```

**复杂度**：O(n log maxVal)

---

#### [2772. 使数组所有元素等于零](./2772/2772.go)

**题目**：能否通过对长度为 k 的子数组减 1，使数组全为 0。

**思路**：差分 + **贪心模拟**

**关键洞察**：
- 位置 `i` 的差值只能通过从 `i` 开始的操作消除
- 必须贪心地消除当前位置差值

```go
// 构建差分数组
diff[0] = nums[0]
diff[i] = nums[i] - nums[i-1]

// 从左到右贪心消除
for i := 0; i+k < n; i++ {
    curSub := diff[i]
    diff[i] = 0
    diff[i+k] += curSub  // 操作影响传递
}

// 检查最后 k 个位置是否都为 0
```

**复杂度**：O(n)

## 四、实战技巧

### 4.1 数组长度

差分数组通常设为 `n+1` 或 `n+2`：
```go
diff := make([]int, n+2)  // 处理 r+1 边界
```

### 4.2 索引处理

- **左闭右闭区间** `[l, r]`：
  ```go
  diff[l] += val
  diff[r+1] -= val
  ```
- **左闭右开区间** `[l, r)`：
  ```go
  diff[l] += val
  diff[r] -= val
  ```

### 4.3 还原时机

所有区间操作完成后，再进行 O(n) 前缀和还原。不要频繁还原。

## 五、对比其他数据结构

| 数据结构 | 区间修改 | 区间查询 | 适用场景 |
|---------|---------|---------|---------|
| 差分数组 | O(1) | O(n) | 多次修改，一次查询 |
| 树状数组 | O(log n) | O(log n) | 修改查询交替 |
| 线段树 | O(log n) | O(log n) | 复杂区间操作 |

## 六、易错点

### 6.1 边界问题
- `diff[r+1]` 可能越界，需要检查或预留空间
- 1094 题中 `to` 是下车点，乘客在 `to` 不占座，所以 `diff[to] -= num` 而非 `diff[to+1]`

### 6.2 负数处理
- 差分数组可能有负数，前缀和时正确处理即可
- 模运算时注意：`(x - 1 + 26) % 26` 而不是 `(x - 1) % 26`

### 6.3 索引起始
- 1109 题航班从 1 开始编号
- 其他题目通常从 0 开始

---

## 七、总结

差分数组的核心价值：**用 O(1) 的区间修改替代 O(n)**

适用条件：
1. 操作是区间统一增减
2. 只关心最终状态
3. 可以接受 O(n) 的最终还原
