# 差分数组 (Difference Array) 学习总结

## 什么是差分数组？

差分数组是一种用于高效处理数组区间修改的数据结构。它通过存储数组相邻元素之间的差值来表示原数组。当需要对原数组的某个区间 `[i, j]` 进行统一增减操作时，我们只需要修改差分数组的两个位置：`diff[i]` 和 `diff[j+1]`。

**基本思想：**

设原数组为 `A`，其差分数组为 `D`。
- `D[0] = A[0]`
- `D[i] = A[i] - A[i-1]` (对于 `i > 0`)

通过差分数组 `D`，可以恢复原数组 `A`：
- `A[0] = D[0]`
- `A[i] = A[i-1] + D[i]` (对于 `i > 0`)

## 差分数组的应用场景

差分数组最常见的应用场景是：

1.  **区间更新**：对数组的 `[left, right]` 区间内的所有元素统一增加 `val` 或减少 `val`。
    - 操作方法：`D[left] += val`，如果 `right + 1` 在数组范围内，则 `D[right+1] -= val`。
    - 这种操作的时间复杂度为 O(1)。

2.  **单点查询（恢复原数组）**：在所有区间修改操作完成后，需要获取原数组某个位置的值，或重建整个原数组。
    - 操作方法：遍历差分数组 `D` 并累加，即可得到原数组 `A`。
    - 重建整个原数组的时间复杂度为 O(N)，N 为数组长度。

**优点：**

- 对于大量的区间修改操作，差分数组的效率远高于直接修改原数组（O(1) vs O(N)）。
- 实现相对简单。

**缺点：**

- 不支持 O(1) 的单点查询，需要 O(N) 重建或 O(N) 累加。如果需要频繁单点查询，可能需要结合树状数组或线段树等更复杂的数据结构。

### **使用差分数组的注意点**

1.  **数组长度与索引边界**：
    - 差分数组的长度通常比原数组多 1，以处理 `right + 1` 的边界情况。
    - 在进行 `D[right+1] -= val` 操作时，务必检查 `right + 1` 是否越界。如果 `right` 是数组的最后一个索引，那么 `right + 1` 就会超出范围，此时不需要进行 `D[right+1]` 的操作。

2.  **初始化**：
    - 如果原数组非空，差分数组的第一个元素 `D[0]` 通常等于 `A[0]`。
    - 如果所有操作都是基于一个全为 0 的初始数组，那么差分数组可以全部初始化为 0。

3.  **还原原数组的时机**：
    - 差分数组的优势在于 O(1) 的区间修改。只有在所有修改操作都完成后，才进行一次 O(N) 的遍历来还原原数组，或者在需要查询某个具体值时进行累加。频繁地在中间过程还原原数组会失去差分数组的效率优势。

4.  **负值处理**：
    - 差分数组中的元素可以是负数。在累加还原原数组时，需要正确处理这些负值。

5.  **与其他算法的结合**：
    - 差分数组常常与其他算法结合使用，例如：
        - **二分查找**：当问题需要寻找满足某种条件的最小/最大值，且该条件可以通过差分数组判断时，可以结合二分查找（如 [2528 最大化城市的最小景气值](file:///Users/silas/gitcode/algorithm/src/leetcode/diff/2528/2528.go) 可能需要）。
        - **贪心算法**：在某些需要最小化或最大化操作的题目中，差分数组可以帮助我们模拟状态，然后结合贪心策略求解。

6.  **离散化**：
    - 如果坐标范围非常大，但有效事件点（如区间起点、终点）相对较少，可以考虑对这些坐标进行离散化处理，将大范围映射到小范围，从而避免创建过大的差分数组。

通过深入理解这些相似点、差异点和注意点，可以更好地掌握差分数组这一强大工具，并将其应用于更广泛的算法问题中。
